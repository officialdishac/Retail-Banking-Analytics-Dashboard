# 📊 Retail Banking Customer & Transaction Analytics Dashboard

## 📌 Project Overview

A **data analytics and business intelligence project** built using **Google Looker Studio**, designed to analyze **retail banking transactions, fraud patterns, revenue performance, and customer segmentation**.

The project integrates insights from a **SQL-based data pipeline** and presents them through an interactive dashboard for business decision-making.

---

## 🎯 Problem Statement

Financial institutions require clear visibility into:
- Transaction behavior patterns
- Fraud-prone categories and transaction types
- Customer value distribution
- Revenue performance metrics

This project addresses these needs by building a **data-driven dashboard for banking analytics and fraud detection insights**.

---

## 🛠️ Tools & Technologies

- Google Looker Studio (Data Visualization / BI)
- SQL (Data extraction & preprocessing)
- Data Modeling (Aggregations & KPIs)
- Spreadsheet / CSV (Dataset handling)

---

## 📊 Dataset Information

- **Total Transactions:** 6,000  
- **Total Revenue:** ₹297,019,836.06  
- **Average Transaction Value:** ₹49,503.31  
- **Fraud Rate:** 4.98%  

📌 Note: The dataset used in Looker Studio is a **sampled version of the original SQL dataset**, optimized for performance and visualization while maintaining data distribution consistency.

---

## 📈 Key Performance Indicators (KPIs)

- Total Transactions
- Total Revenue
- Average Transaction Value
- Fraud Rate

---

## 🔍 Key Insights

### ⚠️ Fraud Analysis
- Fraud rate is approximately **4.98%**, indicating moderate risk exposure.
- Highest fraud concentration observed in:
  - Health
  - Groceries
  - Entertainment
  - Restaurants

### 💳 Transaction Risk Analysis
- Highest fraud contribution by transaction type:
  - Bill Payments
  - Withdrawals
  - Debit Transactions

### 👥 Customer Segmentation
- High Value Customers: **49.1%**
- Medium Value Customers: **31.1%**
- Low Value Customers: **19.9%**

---
